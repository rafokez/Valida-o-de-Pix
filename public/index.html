Documentação da API

Bem-vindo à documentação da API para validação de documentos.

Endpoints Disponíveis

  • 1. Validar Comprovante PIX

    Endpoint: /api/validate

    Método: POST

    Descrição: Valida se a imagem enviada é um comprovante PIX.

    Exemplo de Requisição:

    curl -X POST -F "file=@caminho_para_imagem" http://localhost:3000/api/validate

    Exemplo de Resposta:

    {
  "success": true,
  "isValid": true,
  "patternsMatched": 3,
  "extractedText": "Texto extraído da imagem"
}

  • 2. Validar CTF IBAMA

    Endpoint: /api/validate/ctf-ibama

    Método: POST

    Descrição: Valida se a imagem enviada é um certificado CTF do IBAMA.

    Exemplo de Requisição:

    curl -X POST -F "file=@caminho_para_imagem" http://localhost:3000/api/validate/ctf-ibama

  • 3. Validar Alvará de Funcionamento

    Endpoint: /api/validate/alvara

    Método: POST

    Descrição: Valida se a imagem enviada é um Alvará de Funcionamento.

  • 4. Validar Licença de Operação

    Endpoint: /api/validate/licenca-operacao

    Método: POST

    Descrição: Valida se a imagem enviada é uma Licença de Operação.

  • 5. Validar AVCB Bombeiro

    Endpoint: /api/validate/avcb-bombeiro

    Método: POST

    Descrição: Valida se a imagem enviada é um Auto de Vistoria do Corpo de Bombeiros.

Formato da Requisição

Certifique-se de enviar um arquivo no campo file em formato multipart/form-data.

Erros Comuns

  • 400 - Nenhum arquivo enviado: Certifique-se de incluir o arquivo na requisição.
  • 500 - Erro Interno: Ocorreu um problema ao processar a imagem. Verifique os logs do servidor.
